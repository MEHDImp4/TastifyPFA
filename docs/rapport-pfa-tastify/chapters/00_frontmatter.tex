\chapter*{Résumé}
\addcontentsline{toc}{chapter}{Résumé}
Tastify est une application web full-stack destinée à faciliter la gestion quotidienne d'un restaurant. Le projet répond à plusieurs difficultés fréquemment rencontrées dans les établissements de restauration : coordination entre la salle et la cuisine, suivi des commandes, gestion des stocks, réservations, paiements, fidélisation et analyse des avis clients.

La solution repose sur un backend Django REST Framework, une base de données MySQL, Redis pour le temps réel et les tâches asynchrones, ainsi que deux applications React : une interface back-office destinée au personnel et un portail client. Tastify intègre également un Kitchen Display System permettant de suivre l'avancement des plats en cuisine, un module de paiement avec partage de l'addition et une analyse de sentiment des avis clients via l'API Hugging Face, avec un mécanisme de secours local.

Ce rapport présente le contexte du projet, l'étude de l'existant, la conception UML, l'architecture technique, les principales fonctionnalités réalisées et les tests mis en place pour valider la solution.

\textbf{Mots-clés :} ERP, restaurant, Django, React, MySQL, Redis, Celery, WebSocket, KDS, analyse de sentiment, PFA.

\chapter*{Abstract}
\addcontentsline{toc}{chapter}{Abstract}
Tastify is a full-stack web application designed to support the daily management of a restaurant. The project addresses common operational issues such as communication between waiters and kitchen staff, order tracking, stock management, reservations, payments, loyalty and customer feedback analysis.

The solution is built with a Django REST Framework backend, a MySQL database, Redis for real-time communication and asynchronous tasks, and two React applications: a staff back-office and a customer portal. Tastify also includes a Kitchen Display System, split-bill payment support and sentiment analysis for customer reviews using the Hugging Face API with a local fallback.

This report presents the project context, existing solutions, UML design, technical architecture, implemented features and validation tests.
