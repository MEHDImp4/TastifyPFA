063 | Heading1 | Abstract
064 |  | The restaurant sector faces significant operational challenges, particularly in small and medium-sized establishments where order management, stock tracking, and customer feedback processing are still handled manually. This situation leads to inefficiencies, errors, and missed opportunities for improvement.
065 |  | Tastify is a web-based ERP (Enterprise Resource Planning) application designed specifically for restaurant management. The system centralizes all key operations into a single platform: menu management, order processing, table reservations, stock tracking, and customer feedback collection. It provides dedicated interfaces for four user roles: the manager, the waiter, the kitchen staff, and the customer.
066 |  | A sentiment analysis module has been designed and modelled within the project as a planned extension. Its architecture is based on a Natural Language Processing (NLP) pipeline covering text preprocessing, tokenization, and classification using both classical Machine Learning methods (Naive Bayes, SVM, Logistic Regression, Decision Tree) and a Transformer-based model (DistilBERT from Hugging Face). The full integration of this module into the application constitutes a clearly identified technical perspective.
067 |  | The application is built using Django 5.0 (ASGI/Daphne) for the backend REST API and WebSocket layer, React 18 + TypeScript + Vite for two separate single-page application frontends (backoffice and client portal), MySQL 8 as the primary database, and Redis with Celery for asynchronous task processing. The delivered system is a fully operational, multi-role restaurant ERP encompassing order management, table floor plan, reservations, stock tracking, loyalty programme, HR management, split-bill payments, and a real-time Kitchen Display System.
068 |  | Keywords: Restaurant ERP, Django, React, TypeScript, MySQL, Redis, Celery, WebSocket, Loyalty Programme, Kitchen Display System, NLP, Sentiment Analysis (perspective).

070 | Heading2 | 1. Contexte
071 |  | Le secteur de la restauration reste l'un des domaines les moins touches par la transformation numerique, surtout au niveau des petites et moyennes structures. La plupart des restaurants gerent encore leurs commandes sur papier, coordonnent la salle et la cuisine verbalement, et ne disposent d'aucun outil centralise pour suivre leurs activites. Cette situation genere des erreurs, des pertes de temps et une difficulte a analyser les performances du restaurant.
072 |  | Par ailleurs, les avis clients representent une source d'information precieuse mais souvent inexploitee. Un restaurateur qui recoit des dizaines de commentaires par semaine n'a generalement pas le temps de tous les lire et d'en tirer des conclusions utiles. L'analyse automatique des sentiments, qui consiste a classer ces avis en positifs, negatifs ou neutres a l'aide de techniques d'intelligence artificielle, offre une reponse concrete a ce probleme.
073 |  | C'est dans ce contexte que le projet Tastify a ete developpe. L'objectif est de proposer une application web de gestion de restaurant qui centralise les principales operations du quotidien, tout en integrant un module d'analyse des sentiments pour aider le gerant a mieux comprendre les retours de ses clients. Tastify n'est pas une solution qui cherche a tout revolutionner, mais plutot a repondre a des besoins reels et concrets, avec des technologies accessibles.

271 | Heading1 | Chapitre 3 : Conception du module NLP – Fondements théoriques
272 | Heading2 | Introduction
273 |  | Ce chapitre présente la base théorique sur laquelle repose la conception du module d’analyse des sentiments de Tastify. Ce module a été modélisé et architecturé dans le cadre du projet ; son intégration opérationnelle constitue une perspective d’évolution identifiée. On y abordera les notions de sentiment et d’opinion, le traitement du langage naturel (NLP), l’analyse des sentiments en tant que tâche IA, et une revue des principales méthodes existantes qui ont guidé nos choix de conception.
274 | Heading2 | 1. Sentiments et opinions
275 |  | Un sentiment est une reaction affective qu'une personne exprime par rapport a une experience vecue. Dans le contexte d'un restaurant, il peut s'agir de la satisfaction vis-a-vis d'un plat, de la qualite du service ou de l'ambiance generale. Une opinion, quant a elle, est une position exprimee sur un sujet precis, qui peut etre positive, negative ou neutre.
276 |  | Les avis clients representent une source importante d'informations pour les restaurateurs. Un commentaire comme 'Le poulet etait trop sec, mais le service etait rapide' contient deux opinions : une negative sur la qualite du plat et une positive sur le service. Ces informations, si elles sont analysees correctement, permettent au gerant d'identifier les points a ameliorer et de valoriser les aspects apprecies par les clients.
277 |  | Dans Tastify, les avis clients sont collectes directement dans l'application. L'analyse automatique de ces avis permet d'obtenir rapidement une vision d'ensemble de la satisfaction des clients, sans que le gerant ait a lire chaque commentaire individuellement.
278 | Heading2 | 2. Traitement du langage naturel - NLP
279 |  | Le traitement du langage naturel (NLP, de l'anglais Natural Language Processing) est un domaine de l'intelligence artificielle qui s'interesse a l'interaction entre les machines et le langage humain. Son objectif est de permettre aux ordinateurs de comprendre, d'interpreter et de generer du texte de maniere utile et coherente.
280 |  | Le NLP est utilise dans de nombreuses applications du quotidien :
281 | ListParagraph | La traduction automatique (comme Google Traduction)
282 | ListParagraph | Les assistants virtuels et chatbots
283 | ListParagraph | Le resume automatique de textes
284 | ListParagraph | La classification de documents et d'e-mails
285 | ListParagraph | L'analyse des avis et commentaires en ligne
286 |  | Dans le cadre de Tastify, le NLP est utilise pour analyser automatiquement les commentaires laisses par les clients et en extraire le sentiment general : positif, negatif ou neutre.
287 | Heading2 | 3. Analyse des sentiments
288 |  | L'analyse des sentiments, aussi appelee opinion mining, est une tache du NLP qui consiste a identifier et classer les opinions exprimees dans un texte. Elle attribue a un texte donne une polarite qui peut etre :
289 | ListParagraph | Positive : le client est satisfait (exemple : 'Le plat etait delicieux')
290 | ListParagraph | Negative : le client est mecontent (exemple : 'Le service etait trop lent')
291 | ListParagraph | Neutre : le commentaire est factuel ou ambigu (exemple : 'J'ai commande une salade')
292 |  | Certains systemes vont plus loin et attribuent un score numerique (par exemple de 1 a 5 etoiles) plutot qu'une simple categorie.
293 | Heading3 | 3.1. Domaines d'applications de l'analyse des sentiments
294 |  | L'analyse des sentiments est utilisee dans de nombreux secteurs :
295 | ListParagraph | Commerce et e-commerce : analyse des avis produits sur les sites de vente en ligne.
296 | ListParagraph | Restauration : classement des plats et du service selon les avis clients (cas de Tastify).
297 | ListParagraph | Reseaux sociaux : detection de tendances et d'opinions sur des sujets precis.
298 | ListParagraph | Marketing : mesure de la perception d'une marque ou d'une campagne publicitaire.
299 | ListParagraph | Support client : classification automatique des tickets selon leur ton ou urgence.
300 | ListParagraph | Politique : analyse de l'opinion publique sur les reseaux sociaux.
301 |  | Dans Tastify, l'analyse des sentiments est directement liee aux avis clients. Le module traite automatiquement chaque commentaire des qu'il est soumis, et le resultat est affiche dans le tableau de bord du gerant.
302 | Heading3 | 3.2. Niveaux d'analyse des sentiments
303 |  | Il existe trois niveaux d'analyse des sentiments :
304 | ListParagraph | Niveau document : le sentiment est analyse sur l'ensemble d'un texte. Par exemple, un avis client complet est classe comme positif ou negatif. C'est le niveau le plus courant.
305 | ListParagraph | Niveau phrase : chaque phrase du texte est analysee separement. Cela permet de detecter des opinions contradictoires dans un meme commentaire.
306 | ListParagraph | Niveau aspect : l'analyse porte sur des aspects precis mentionnes dans le texte, comme le gout du plat, le prix, ou la rapidite du service. Ce niveau est le plus fin mais aussi le plus complexe.
307 |  | Dans le cadre de Tastify, l'analyse se fait principalement au niveau document, ce qui correspond au besoin principal : donner une indication rapide sur la polarite globale d'un avis client.

319 |  | Naive Bayes, SVM, Logistic Regression, Decision Tree

321 |  | Necessite des donnees d'entrainement, feature engineering manuel
322 |  | Deep Learning
323 |  | RNN, LSTM, CNN

331 |  | Les premieres approches d'analyse des sentiments reposaient sur des dictionnaires de mots positifs et negatifs. Si un texte contient plus de mots positifs que negatifs, il est classe comme positif. Cette methode est simple mais limitee : elle ne prend pas en compte le contexte ni la syntaxe.
332 |  | Les methodes de Machine Learning classique, comme Naive Bayes ou SVM, ont ensuite montre de meilleures performances, en apprenant a partir de textes deja etiquetes. Ces modeles restent utilises car ils sont rapides et faciles a interpreter.
333 |  | L'arrivee du Deep Learning, avec les reseaux de neurones recurrents (RNN) et convolutifs (CNN), a permis de mieux capturer les relations entre les mots dans une phrase. Les LSTM notamment ont ete populaires pour les taches de NLP avant l'emergence des Transformers.
334 |  | Aujourd'hui, les modeles Transformers comme BERT (Bidirectional Encoder Representations from Transformers, Devlin et al., 2018) representent l'etat de l'art en analyse des sentiments. Ils comprennent le contexte complet d'un texte en lisant les mots simultanement dans les deux sens. DistilBERT est une version allegee de BERT qui conserve 97% des performances avec 40% moins de parametres, ce qui le rend plus adapte a un deploiement dans une application comme Tastify.
335 | Heading1 | Chapitre 4 : Pipeline conçu pour l’analyse des sentiments
336 | Heading2 | Introduction
337 |  | Ce chapitre décrit le pipeline d’analyse des sentiments tel qu’il a été conçu et modélisé dans le cadre du projet Tastify. Les étapes présentées — collecte des données, prétraitement du texte, vectorisation et classification — constituent la feuille de route de l’intégration NLP. Chaque étape est documentée avec des exemples concrets pour faciliter la mise en œuvre future.
338 | Heading2 | 1. Collecte de donnees
339 |  | Les donnees utilisees dans le module d'analyse des sentiments proviennent principalement des avis que les clients laissent directement dans l'application Tastify. Chaque avis est associe a un plat et contient un texte libre ainsi qu'une note numerique.
340 |  | Pour entrainer et tester les modeles de classification, il est possible d'utiliser des jeux de donnees publics d'avis de restaurants, comme des datasets issus de plateformes d'avis en ligne. Ces donnees permettent de calibrer les modeles avant de les appliquer aux avis reels de l'application.
341 |  | Dans le cadre de ce projet, les donnees peuvent provenir des avis laisses par les utilisateurs ou d'un jeu de donnees d'avis clients utilise pour tester les modeles. Les donnees sont stockees en base de donnees et traitees a chaque soumission d'un nouvel avis.
342 | Heading2 | 2. Pretraitement de la base de donnees
343 |  | Avant d'appliquer un modele de classification, le texte brut doit etre nettoye et normalise. Cette etape est essentielle pour reduire le bruit et ameliorer la qualite de la classification. Voici les principales transformations appliquees.
344 | Heading3 | 2.1. Tokenization
345 |  | La tokenization consiste a decouper un texte en unites elementaires appelees tokens. Ces tokens peuvent etre des mots, des sous-mots ou meme des caracteres selon le modele utilise.
346 |  | Exemple :
347 |  | Phrase originale : "Le plat etait tres bon !"
348 |  | Tokens : ["Le", "plat", "etait", "tres", "bon", "!"]
349 | Heading3 | 2.2. Suppression des mots vides
350 |  | Les mots vides (stopwords) sont des mots tres frequents qui ne portent pas de sens particulier, comme 'le', 'la', 'de', 'un', 'et', 'a', 'est', etc. Dans les methodes classiques, ces mots sont souvent supprimes pour alleger le traitement et ameliorer les performances.
351 |  | Il est important de noter que pour les modeles Transformers comme BERT, cette etape n'est generalement pas appliquee. BERT utilise le contexte complet de la phrase pour comprendre le sens, et supprimer des mots pourrait perturber cette comprehension contextuelle.
352 | Heading3 | 2.3. Stemming
353 |  | Le stemming consiste a reduire les mots a leur racine commune, quelle que soit leur forme grammaticale. L'objectif est de regrouper les variantes d'un meme mot sous une seule representation.
354 |  | Exemple :
355 |  | "manger", "mange", "mangeons", "mangera"  -->  "mang"
356 |  | Cette technique est utile pour les methodes classiques, mais moins pertinente pour BERT, qui integre deja une tokenization avancee prenant en compte les variations morphologiques des mots.
357 | Heading2 | 3. Word Embedding - Vectorisation du texte
358 |  | Les modeles de Machine Learning ne peuvent pas traiter directement du texte. Il faut transformer les mots en representations numeriques (vecteurs). C'est ce que realisent les techniques de word embedding presentees ci-dessous.
359 | Heading3 | 3.1. Term Frequency - Inverse Document Frequency (TF-IDF)
360 |  | TF-IDF est une methode statistique qui mesure l'importance d'un mot dans un document par rapport a un corpus de documents. Un mot frequent dans un document mais rare dans les autres aura un score eleve, ce qui permet d'identifier les mots vraiment significatifs pour ce document. Cette methode est souvent utilisee en combinaison avec des classifieurs comme Naive Bayes ou SVM.
361 | Heading3 | 3.2. Word2Vec
362 |  | Word2Vec est un modele neuronal qui apprend a representer chaque mot sous forme d'un vecteur dense dans un espace multidimensionnel. Sa particularite est que les mots ayant des sens proches ont des vecteurs proches dans cet espace. Par exemple, les vecteurs de 'bon' et 'excellent' seront similaires, tandis que 'bon' et 'mauvais' seront eloignes.
363 | Heading3 | 3.3. FastText
364 |  | FastText est une amelioration de Word2Vec developpee par Facebook. Sa particularite est qu'il prend en compte les sous-mots (n-grammes de caracteres), ce qui le rend plus robuste face aux fautes d'orthographe et aux variations morphologiques. Un mot inconnu peut ainsi etre represente a partir de ses parties.
365 | Heading3 | 3.4. GloVe
366 |  | GloVe (Global Vectors for Word Representation) est une methode basee sur les statistiques globales de cooccurrence des mots dans un corpus. Elle construit des vecteurs de mots en analysant combien de fois des mots apparaissent ensemble dans l'ensemble du corpus, ce qui permet de capturer des relations semantiques globales.
367 | Heading3 | 3.5. BERT
368 |  | BERT (Bidirectional Encoder Representations from Transformers) represente une avancee majeure dans le domaine du NLP. Contrairement aux methodes precedentes qui assignent un vecteur fixe a chaque mot, BERT genere des representations contextuelles : le meme mot peut avoir des representations differentes selon le contexte de la phrase.
369 |  | BERT lit le texte dans les deux sens simultanement grace au mecanisme d'attention, ce qui lui permet de comprendre des nuances complexes comme la negation ou l'ironie.
370 |  | Dans Tastify, un modele base sur DistilBERT (version allegee de BERT) est utilise pour analyser automatiquement les commentaires clients et produire une classification positive/negative/neutre.
371 | Heading2 | 4. Classification
372 |  | La classification est l'etape finale du pipeline d'analyse des sentiments. Elle consiste a attribuer une etiquette (positif, negatif, neutre) a un texte donne, en se basant sur les representations numeriques produites lors de l'etape de vectorisation.
373 | Heading3 | 4.1. Methodes classiques
374 |  | Les methodes basees sur des dictionnaires consistent a associer un score a chaque mot selon sa polarite (positif, negatif, neutre), puis a calculer un score global pour le texte. Ces methodes sont rapides et ne necessitent pas de donnees d'entrainement, mais elles ne gerent pas bien le contexte ni les expressions idiomatiques.
375 | Heading3 | 4.2. Methodes Machine Learning
376 | Heading4 | 4.2.1. Methodes supervisees
377 |  | Les methodes supervisees apprennent a classer des textes a partir d'exemples deja etiquetes. Le modele est entraine sur un ensemble de donnees ou chaque texte est associe a un sentiment connu, puis il est teste sur de nouvelles donnees.
378 | Heading4 | 4.2.1.1. Naive Bayes
379 |  | Naive Bayes est un classifieur probabiliste base sur le theoreme de Bayes. Il calcule la probabilite qu'un texte appartienne a chaque classe (positif, negatif, neutre) en se basant sur la frequence des mots. Malgre sa simplicite, il donne souvent de bons resultats pour la classification de texte et est tres rapide a entrainer.
380 | Heading4 | 4.2.1.2. Bayesian Network
381 |  | Un reseau bayesien est un modele probabiliste qui represente les dependances entre plusieurs variables sous forme d'un graphe acyclique oriente. Il est plus complexe que Naive Bayes car il prend en compte les relations entre les variables, mais il est aussi plus couteux en calcul.
382 | Heading4 | 4.2.1.3. Maximum Entropy (MaxEnt)
383 |  | Le modele a maximum d'entropie (aussi appele regression logistique multinomiale) cherche a predire la classe d'un texte sans faire d'hypotheses trop fortes sur la distribution des donnees. Il est souvent utilise en NLP car il gere bien les features redondantes.
384 | Heading4 | 4.2.1.4. Support Vector Machine (SVM)
385 |  | SVM est un algorithme de classification qui cherche a trouver la frontiere optimale (hyperplan) separant les classes dans l'espace des features. Il est particulierement efficace pour la classification de texte avec TF-IDF, ou les vecteurs sont de grande dimension.
386 | Heading4 | 4.2.1.5. Decision Tree
387 |  | Un arbre de decision est un modele qui prend des decisions successives en testant les valeurs de differentes features. Pour la classification de texte, chaque noeud de l'arbre peut representer la presence ou l'absence d'un mot cle. Ce modele est facile a interpreter et a visualiser.
388 | Heading4 | 4.2.2. Methodes Deep Learning
389 | Heading4 | 4.2.2.1. Reseaux de neurones
390 |  | Les reseaux de neurones artificiels s'inspirent du fonctionnement du cerveau humain. Ils sont composes de couches de neurones interconnectes qui transforment progressivement les donnees d'entree pour produire une prediction. Appliques au texte, ils permettent d'apprendre des representations complexes.
391 | Heading4 | 4.2.2.2. Reseaux de neurones convolutifs (CNN)
392 |  | Bien que les CNN soient surtout connus pour le traitement d'images, ils peuvent aussi etre appliques au texte. Les filtres convolutifs permettent de detecter des motifs locaux (n-grammes) dans une sequence de mots, ce qui les rend utiles pour capturer des expressions importantes dans un avis.
393 | Heading4 | 4.2.2.3. Reseaux de neurones recurrents (RNN)
394 |  | Les RNN sont concus pour traiter des donnees sequentielles comme le texte. Ils maintiennent un etat interne (memoire) qui evolue au fil de la lecture des mots, ce qui leur permet de capturer des dependances entre des mots eloignes dans la phrase. Les variantes LSTM et GRU ameliorent encore la capacite a gerer de longues sequences.
395 | Heading4 | 4.2.2.4. Transformers - BERT, DistilBERT
396 |  | Les Transformers representent l'architecture dominante en NLP depuis 2018. Leur mecanisme d'attention (self-attention) permet au modele de peser l'importance de chaque mot par rapport a tous les autres mots de la phrase, simultanement. BERT utilise cette architecture en mode bidirectionnel, ce qui lui permet de comprendre le contexte complet d'un mot.
397 |  | DistilBERT est une version compressee de BERT : il est 40% plus leger, 60% plus rapide, tout en conservant 97% des performances de BERT sur la plupart des taches NLP. C'est un choix adapte pour une application comme Tastify ou les ressources de calcul sont limitees.
398 | Heading2 | Conclusion du chapitre 4
399 |  | Ce chapitre a decrit le pipeline complet d'analyse des sentiments tel qu'il est mis en oeuvre dans Tastify. Les etapes cles sont : la collecte des avis clients, le pretraitement du texte (nettoyage, tokenization, stemming), la vectorisation (TF-IDF, Word2Vec, BERT), et la classification par un modele de Machine Learning ou par un modele Transformer. Le resultat de cette analyse est ensuite sauvegarde en base de donnees et mis a disposition du gerant via le tableau de bord. Le chapitre suivant presentera les interfaces realisees et les captures d'ecran de l'application.
