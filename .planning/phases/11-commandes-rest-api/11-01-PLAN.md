# Plan 11-01: Serializers & ViewSet Foundation

## Goal
Implement nested serializers and the core `CommandeViewSet` with atomic creation support and ownership-based filtering.

## Proposed Changes

### backend/apps/commandes/serializers.py
- Implement `CommandeLigneSerializer`.
- Implement `CommandeSerializer` with:
  - `lignes` as a nested `CommandeLigneSerializer`.
  - Overridden `create()` using `transaction.atomic()`.
  - Automatic `serveur` assignment from `self.context['request'].user`.

### backend/apps/commandes/views.py
- Implement `CommandeViewSet` extending `viewsets.ModelViewSet`.
- Implement `get_queryset()` to filter by `serveur=user` for non-GERANTs.
- Implement RBAC in `get_permissions()`: `IsAuthenticated` for all, `IsGerant` for some (if applicable).

### backend/tastify_backend/api_router.py
- Register `commandes` endpoint.

### backend/apps/commandes/tests/test_api.py
- Add `CommandeAPITestCase`.
- Test atomic creation with multiple lines.
- Test RBAC (Serveur vs Gérant visibility).
- Test ownership filtering.

## Verification Plan

### Automated Tests
- `python backend/manage.py test apps.commandes.tests.test_api`

### Manual Verification
- `POST /api/commandes/` with valid/invalid data using `curl` or Postman.
- Check database state after creation.
