  <name>Task 2: Manual UAT — audio bell + glow + end-to-end fire flow</name>
  <what-built>
    The complete Phase 16 "Manual Fire" workflow is wired end-to-end:
    - Backend: signals fire orchestration only on EN_COURS->EN_CUISINE, KDS queryset filter excludes drafts, PATCH ownership guard rejects non-owners with 403.
    - Frontend (Salle): "Tout Envoyer en Cuisine" button on OrderingPage when an order is EN_COURS and owned.
    - Frontend (KDS): audio bell on EN_CUISINE arrival, green pulse glow on the new TicketCard for 10 seconds, draft EN_COURS orders never appear on the rail.
  </what-built>
  <how-to-verify>
**Setup (one-time per UAT session):**

1. Ensure Docker stack is running: `docker compose up -d backend frontend-back-office redis celery-worker daphne`. Wait for all services to be healthy.
2. Open two browser windows side by side:
   - Window A: Salle — `http://localhost:3000/staff/login`. Log in as a SERVEUR account (e.g. `serveur` / `password123` from seed).
   - Window B: KDS — second browser session, `http://localhost:3000/staff/login`. Log in as a CUISINIER account (e.g. `cuisinier` / `password123`). Navigate to the KDS page.
3. Click anywhere in Window B (KDS) at least once to satisfy the audio autoplay gesture requirement.

**Verification 1 — KDS rail hides EN_COURS drafts (P16-BE-04 visual confirmation):**

4. In Window A (SERVEUR), open a table page and add at least 2 dishes from different categories (so the JIT orchestrator has something to compute). Click the cart "Validate" / "Submit" button to create the EN_COURS order.
5. Observe Window B (KDS): the new order MUST NOT appear on the rail. The "Cuisine Vide" empty state should still show (or, if other fired tickets exist, the new draft simply does not appear).
6. **Pass criterion:** No new TicketCard for the just-created EN_COURS order is visible on the KDS rail.

**Verification 2 — Fire button + PATCH + audio + glow (P16-FE-Audio + P16-FE-Glow + end-to-end):**

7. In Window A, observe the "Tout Envoyer en Cuisine" button is now visible above "Clôturer et Encaisser".
8. Click "Tout Envoyer en Cuisine".
9. Within 1-2 seconds, in Window B (KDS):
   - **Audio:** A bell sound plays (or, if using the silent placeholder MP3 from Plan 02, no sound is heard but no console error appears in DevTools either). If using a production-quality replacement bell, you should clearly hear the bell.
   - **Glow:** The new TicketCard appears with a pulsing green outer glow visibly different from neighbouring tickets.
10. Wait approximately 10 seconds. The green glow MUST stop pulsing and the TicketCard appears like any other.
11. **Pass criteria:**
   - Bell played (or autoplay-blocked silently — verify DevTools console shows no unhandled rejection from `audio.play()`)
   - Glow was clearly visible for ~10 seconds
   - Glow stopped after ~10 seconds without manual page refresh

**Verification 3 — Status regression hidden from KDS (P16-BE-04):**

12. In Window B, leave the order on the KDS rail. (Phase 17 will mark it PRETE; Phase 16 only locks visibility.)
13. In Window A, the order's "Tout Envoyer en Cuisine" button is GONE (statut is now EN_CUISINE). The "Clôturer et Encaisser" button remains.
14. **Pass criterion:** Salle UI correctly hides the fire button after the PATCH succeeds; closeOrder remains.

**Verification 4 — Non-owner 403 (T-16-01 visual smoke test):**

15. In Window A, log out and log in as a DIFFERENT SERVEUR account (e.g. `serveur2`).
16. Navigate to the SAME table from steps 4. The activeOrder banner shows "Cette table est gérée par {original serveur}" — readonly view.
17. Use DevTools to verify no fire button is rendered (the `isOwnOrder` guard hides it).
18. (Optional) From DevTools network tab, manually fire `await fetch('/api/commandes/{id}/', {method:'PATCH', ...})`. Backend MUST respond 403 with `{"error":"Vous n'êtes pas autorisé à modifier cette commande."}`.
19. **Pass criterion:** Either UX hides the button (sufficient for visual UAT) OR a manual PATCH attempt returns 403.
