# Phase 17-01: Backend CommandeLigne Endpoint

## 1. Goal
Introduce a direct endpoint to update `CommandeLigne` status, allowing the Kitchen staff to mark items as `PRET`.

## 2. Strategy
- Implement `CommandeLigneViewSet` in `apps/commandes/views.py`.
- Enforce strict role-based transitions:
  - `CUISINIER` or `GERANT` can mark as `EN_PREPARATION` or `PRET`.
  - `SERVEUR` (owner) can mark as `SERVI`.
- Register the viewset as `commandelignes` in `api_router.py`.

## 3. Implementation Plan

### Step 1: Add CommandeLigneViewSet
- Update `app/backend/apps/commandes/views.py`.
- Inherit from `viewsets.GenericViewSet` and `mixins.UpdateModelMixin`.
- Implement `partial_update` with status validation.

### Step 2: Register in API Router
- Update `app/backend/tastify_backend/api_router.py`.

### Step 3: Add Automated Tests
- Create `app/backend/apps/commandes/tests/test_ligne_api.py`.
- Test `CUISINIER` marking a line as `PRET`.
- Test `SERVEUR` (owner) marking a line as `SERVI`.
- Test permission denials.

## 4. Verification Plan

### Automated Tests
- Run `pytest app/backend/apps/commandes/tests/test_ligne_api.py`.
- Run full suite `pytest app/backend/apps/commandes/tests/`.

### Manual Verification
- None (covered by automated tests).
