# Phase 23-01 Plan: Reservations Model & Overlap Logic

## Goal
Implement the `Reservation` model in a new `reservations` app and build the core availability validation logic.

## Tasks
1. **Scaffold App**: `python manage.py startapp reservations`.
2. **Define Model**: Implement `Reservation` in `apps/reservations/models.py`.
    - Fields as researched.
    - Status: `PENDING`, `CONFIRMED`, `CANCELLED`, `COMPLETED`, `NO_SHOW`.
3. **Availability Logic**:
    - Implement a `check_availability` method on the `Table` model or a service class.
    - Handle the 15-minute buffer.
4. **Model Validation**:
    - Override `clean()` to prevent overlapping reservations and capacity mismatches.
5. **Migrations**: Create and apply migrations.

## Verification
- **Unit Tests**:
    - `test_no_overlap_allowed`: Ensure two reservations cannot overlap on the same table.
    - `test_capacity_validation`: Ensure `nombre_personnes` <= `table.capacite`.
    - `test_buffer_logic`: Ensure the 15-minute buffer is respected.
