# Phase 25 Plan 03: Salle/Map Integration & Status Lifecycle

## Goal
Integrate reservation awareness into the Table Map and finalize the status lifecycle (Check-in).

## Tasks

### 1. Enriched Table Serializer
- **File**: `app/backend/apps/tables/serializers.py`
- **Action**: Add `prochaine_reservation` field (read-only).
- **Logic**: Find the nearest active reservation for today that hasn't started yet.

### 2. Map View Enhancements
- **File**: `app/frontend/backoffice/src/pages/Staff/Map/MapView.tsx`
- **Action**: 
    - Update the table info panel to show `Prochaine réservation`.
    - If a reservation is current (`statut_effectif === 'RESERVEE'`), show client name and "Arrivé" button.
    - Button "Arrivé" PATCHes reservation status to `PRESENTE`.

### 3. Automatic Status Transitions (Optional/Future)
- **Concept**: If a table is occupied (`OCCUPEE` by an order), should we automatically mark the reservation as `PRESENTE`?
- **Decision**: Keep it manual for now to allow staff to confirm which client arrived.

## Verification
- **Automated**:
    - `pytest app/backend/apps/tables/tests/test_serializers.py` (Verify `prochaine_reservation`).
    - `vitest app/frontend/backoffice/src/pages/Staff/Map/MapView.test.tsx`.
- **Manual**:
    - Select a reserved table in `MapView`, verify client info is shown.
    - Click "Arrivé" and verify reservation status updates in Back-Office list.
