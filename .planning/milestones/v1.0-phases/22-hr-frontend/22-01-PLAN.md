# Phase 22: HR Frontend

## Goal
Implement the Employee management interface in the Back-Office for the Manager.

## Deliverables
- `app/frontend/backoffice/src/pages/Hr/HrPage.tsx`: Main management page.
- `app/frontend/backoffice/src/pages/Hr/components/EmployeeTable.tsx`: List of employees.
- `app/frontend/backoffice/src/pages/Hr/components/EmployeeModal.tsx`: Form for add/edit.
- `app/frontend/backoffice/src/services/hrService.ts`: API service for HR.
- Navigation: Update Sidebar and App routes.

## Implementation Steps
1. **Service:** Create `hrService.ts` to consume `/api/employes/`.
2. **Page Structure:** Basic layout with title and "Add Employee" button.
3. **Table:** Display salary, position, and status.
4. **Modal:** Form with validation (required fields, CIN format, etc.).
5. **Soft Delete:** Integration of the deletion (deactivation) trigger.
6. **Routes:** Register `/hr` route in `App.tsx`.
7. **Sidebar:** Point "RH" link to `/hr`.

## Verification Strategy
- [ ] List employees successfully.
- [ ] Create employee (check user creation).
- [ ] Edit employee details.
- [ ] Delete employee (verify "is_active" becomes false in users list).
- [ ] Verify RBAC (only Gerant sees the menu).
