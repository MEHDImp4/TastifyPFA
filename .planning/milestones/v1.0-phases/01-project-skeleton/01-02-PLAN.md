**Step B — `backend/Dockerfile`:** Create with this exact content. The `apt-get install` block is MANDATORY before `pip install` — without it `mysqlclient` build fails with `mysql_config not found` (Pitfall 1 in RESEARCH).

```dockerfile
FROM python:3.12-slim

ENV PYTHONDONTWRITEBYTECODE=1 \
    PYTHONUNBUFFERED=1 \
    PIP_NO_CACHE_DIR=1

RUN apt-get update && apt-get install -y --no-install-recommends \
        default-libmysqlclient-dev \
        gcc \
        pkg-config \
        python3-dev \
        curl \
    && rm -rf /var/lib/apt/lists/*

WORKDIR /app

COPY requirements.txt /app/requirements.txt
RUN pip install --upgrade pip && pip install -r requirements.txt

COPY . /app

EXPOSE 8000

CMD ["daphne", "-b", "0.0.0.0", "-p", "8000", "--proxy-headers", "tastify_backend.asgi:application"]
```

The `--proxy-headers` flag is required because Nginx proxies to Daphne (Plan 04) — without it Daphne won't read `X-Forwarded-For` correctly (T-1-08 mitigation).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| nginx ↔ daphne (intra-docker) | HTTP requests cross from Nginx container into backend container; `--proxy-headers` makes X-Forwarded-* trusted |
| .env file ↔ Django process | Secrets enter the process via decouple.config() |
| public client ↔ /api/ | Currently only /api/ health view; no auth (Phase 1 has no business logic) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-05 | Information Disclosure | `DEBUG=True` accidentally enabled in prod | mitigate | `base.py` defaults `DEBUG=False`; `prod.py` explicitly sets `DEBUG=False`; only `dev.py` enables it. Operational hygiene + ASVS V14.3.2. |
| T-1-06 | Spoofing | Open CORS in production | mitigate | `CORS_ALLOW_ALL_ORIGINS = True` is ONLY in `dev.py`; `prod.py` uses `CORS_ALLOWED_ORIGINS` whitelist (empty placeholder). Acceptance criterion verifies `prod.py` does not contain `CORS_ALLOW_ALL_ORIGINS`. ASVS V13.1.5. |
| T-1-07 | Information Disclosure | `mysqlclient` C extension build leaks build deps into final image | accept | Phase 1 uses single-stage Dockerfile (simpler debugging). Multi-stage refactor deferred to Phase 35 (load testing & optimization). Image size only — no secret exposure. |
| T-1-08 | Spoofing | Daphne reads X-Forwarded-For without validation, spoofable IP in logs | mitigate | `--proxy-headers` flag in CMD makes Daphne require the headers come from a trusted upstream (Nginx in same Docker network). Future Phase 35 may add `FORWARDED_ALLOW_IPS`. ASVS V14.4.7. |
| T-1-09 | Tampering | mysqlclient package supply-chain attack | accept | Versions pinned to specific releases verified against pip registry in 01-RESEARCH.md. Lock file (`pip freeze`) deferred to a later infra phase. |
| T-1-10 | Denial of Service | Health endpoint trivially DoS-able | accept | Phase 1 dev infra only; no rate limiting. Real DoS protection is Phase 35 concern. |
</threat_model>
