<objective>
Implement Django signals to monitor `stock_actuel` against `seuil_alerte` and broadcast real-time WebSocket events to the `GERANT` when a threshold is crossed downwards.

Purpose: Provide real-time alerts for low stock levels.
Output: Signals capturing low stock threshold crossing and triggering WebSocket broadcasts.
</objective>

<task type="auto">
  <name>Task 1: Real-Time Alerts via Signals</name>
  <files>app/backend/apps/stock/signals.py, app/backend/apps/stock/apps.py</files>
  <action>
    1. Create `apps/stock/signals.py`.
    2. Connect `pre_save` to `Ingredient`. If the instance exists (`instance.pk`), store the old value: `instance._old_stock = sender.objects.get(pk=instance.pk).stock_actuel`. (Must use `pre_save` to cache `_old_stock` to prevent WebSocket alert spam).
    3. Connect `post_save` to `Ingredient`. Check if the stock is newly low: `is_now_low = instance.stock_actuel &lt;= instance.seuil_alerte`.
    4. Check if it was previously low: `was_low = getattr(instance, '_old_stock', None) is not None and instance._old_stock &lt;= instance.seuil_alerte`.
    5. If `is_now_low` and not `was_low` (or it's a new item created already below threshold), invoke `broadcast_staff_event(event_type="stock.alert", payload={...})` to push the warning (per D-06).
    6. Register these signals in `apps/stock/apps.py` within the `ready()` method.
  </action>
  <verify>
    <automated>docker compose exec backend python manage.py check</automated>
  </verify>
  <done>Signals are created and registered in the apps config without errors.</done>
</task>
