  <action>
    1. Open `app/backend/apps/stock/views.py`. Below the existing `IngredientViewSet`, add:

    ```python
    class PlatIngredientViewSet(ModelViewSet):
        serializer_class = PlatIngredientSerializer
        permission_classes = [IsAuthenticated, IsGerant]
        queryset = PlatIngredient.objects.select_related('plat', 'ingredient').all()
    ```

    Import `PlatIngredient` and `PlatIngredientSerializer` at the top of the file alongside the
    existing imports. Do not remove or modify `IngredientViewSet`.

    2. Open `app/backend/apps/stock/urls.py`. Register the new viewset on the existing router:

    ```python
    router.register(r'plat-ingredients', PlatIngredientViewSet, basename='platIngredient')
    ```

    This produces endpoints:
    - `GET  /api/stock/plat-ingredients/`         — list all links
    - `POST /api/stock/plat-ingredients/`          — create link (GERANT only)
    - `GET  /api/stock/plat-ingredients/{id}/`     — retrieve
    - `PUT  /api/stock/plat-ingredients/{id}/`     — full update (GERANT only)
    - `PATCH /api/stock/plat-ingredients/{id}/`    — partial update (GERANT only)
    - `DELETE /api/stock/plat-ingredients/{id}/`   — delete (GERANT only)

    3. Create `app/backend/apps/stock/admin.py` with the following content, mirroring the pattern
    in `apps/menu/admin.py`:

    ```python
    from django.contrib import admin
    from .models import Ingredient, PlatIngredient


    @admin.register(Ingredient)
    class IngredientAdmin(admin.ModelAdmin):
        list_display = ['nom', 'unite_mesure', 'stock_actuel', 'seuil_alerte', 'est_active', 'created_at']
        list_filter = ['unite_mesure', 'est_active']
        search_fields = ['nom']


    @admin.register(PlatIngredient)
    class PlatIngredientAdmin(admin.ModelAdmin):
        list_display = ['plat', 'ingredient', 'quantite_requise']
        list_filter = ['ingredient__unite_mesure']
        search_fields = ['plat__nom', 'ingredient__nom']
    ```
