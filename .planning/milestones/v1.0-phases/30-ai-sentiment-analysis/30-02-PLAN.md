# Phase 30, Plan 02: DRF API Endpoints

## Goal
Expose REST API endpoints for Clients to submit reviews and Gerants to view them.

## Context
Clients interact with the system via the Portail Client SPA. When they submit a review, the backend should save it and immediately trigger the sentiment analysis task in the background, returning a 201 Created without blocking.

## Steps
1. Create `AvisSerializer` in `apps/avis/serializers.py` exposing all relevant fields.
2. Create `AvisViewSet` in `apps/avis/views.py` inheriting from `ModelViewSet` or `GenericViewSet`.
3. Configure RBAC: 
   - `CLIENT` role can create reviews and read their own reviews.
   - `GERANT` role can read all reviews (read-only).
4. Override `perform_create` on `AvisViewSet`. After calling `serializer.save()`, dispatch the Celery task: `analyze_review_sentiment.delay(instance.id)`.
5. Register the `avis` router in `app/backend/tastify_backend/urls.py` (e.g., `/api/avis/`).
6. Write integration tests for the API to verify the asynchronous task is enqueued on creation.
