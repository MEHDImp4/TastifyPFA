# Phase 30, Plan 01: Models and Celery Infrastructure

## Goal
Create the `avis` Django app, the `Avis` model, and the Celery task for HuggingFace inference.

## Context
We need to capture customer reviews and run sentiment analysis on them. The NLP inference is computationally heavy, so it must run asynchronously in a Celery worker. We also need to avoid Docker image bloat by using the CPU-only version of PyTorch.

## Steps
1. Add dependencies `torch==2.2.2+cpu` (with `--extra-index-url https://download.pytorch.org/whl/cpu`) and `transformers==4.39.3` to `app/backend/requirements.txt`.
2. Create a new Django app `avis` inside `app/backend/apps/`. Add it to `INSTALLED_APPS` in `app/backend/tastify_backend/settings.py`.
3. Create the `Avis` model in `apps/avis/models.py`. It should link to `Plat` (or `Commande`), the user (`settings.AUTH_USER_MODEL`), `commentaire` (text), `note` (1-5 integer), and `sentiment_score` (nullable integer).
4. Create `apps/avis/tasks.py` with a Celery task `analyze_review_sentiment(review_id)`.
5. Implement lazy-loading in the task for `pipeline("sentiment-analysis", model="nlptown/bert-base-multilingual-uncased-sentiment")` globally to save worker memory.
6. The task should fetch the `Avis`, run inference on `commentaire`, parse the result into an integer, and update `sentiment_score`.
7. Write tests for the model and task logic (mocking the HuggingFace pipeline to avoid downloading models during tests).
