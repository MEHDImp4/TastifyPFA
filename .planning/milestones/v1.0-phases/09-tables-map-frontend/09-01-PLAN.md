# Plan 09-01: Table Map Foundation

## Objective
Implement the core SVG map visualization in the Salle UI. Tables will be rendered based on their API data, with a fallback grid layout if coordinates are missing.

## Tasks

### 1. Component: TableItem.tsx
- Specialized SVG component for a single table.
- Props: `Table` object, `onClick`.
- Visuals: Rect or Circle with status-based coloring.
- Animation: `framer-motion` hover and status color transitions.

### 2. Component: TableMap.tsx
- Main SVG container (1000x800 coordinate system).
- Responsive: Viewbox handling.
- Logic: Map over tables and render `TableItem`.
- Grid Fallback: If `pos_x` and `pos_y` are both 0.0, calculate a grid position.

### 3. Page: MapView.tsx
- Fetch tables from `/api/tables/`.
- State: `tables`, `loading`, `error`.
- Polling: Simple 10s interval for status updates (until WS in Phase 13).

### 4. Integration: App.tsx
- Replace the current placeholder view with `MapView`.
- Ensure `AppShell` (if any) or basic layout exists.

## Success Criteria
- [ ] Users see 12 tables in a grid or mapped layout.
- [ ] Colors match status (Green for Libre, Red for Occupee).
- [ ] Tables are clickable.
