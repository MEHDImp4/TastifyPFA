# Phase 25 Plan 01: Backend Enriched API & Real-time Signals

## Goal
Enrich the Reservations API with nested details and real-time WebSocket broadcasts to prepare for the Admin UI.

## Tasks

### 1. Enriched Serializer
- **File**: `app/backend/apps/reservations/serializers.py`
- **Action**: Add `client_details` and `table_details` read-only fields.
- **Fields**:
    - `client_details`: `username`, `first_name`, `last_name`.
    - `table_details`: `numero`, `capacite`.

### 2. WebSocket Signals
- **File**: `app/backend/apps/reservations/signals.py`
- **Action**: Implement `post_save` and `post_delete` signals for `Reservation`.
- **Logic**: Use `transaction.on_commit` to broadcast `reservation_updated` or `reservation_deleted` events to the `staff` group via `core.realtime.broadcast_staff_event`.

### 3. Signal Registration
- **File**: `app/backend/apps/reservations/apps.py`
- **Action**: Import signals in the `ready()` method.

### 4. Admin API Actions (Optional)
- **File**: `app/backend/apps/reservations/views.py`
- **Action**: Verify if explicit actions like `confirmer`, `annuler` are needed. Standard `partial_update` might be sufficient, but explicit actions are better for the "Confirmer" button click.
