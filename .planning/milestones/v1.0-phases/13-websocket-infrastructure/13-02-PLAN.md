<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Auth store -> WebSocket URL | Browser-held access token becomes a query parameter for the socket handshake. |
| WebSocket frame -> React state | Server-sent JSON becomes client state available to downstream components. |
| Network close event -> reconnect timer | Untrusted network failures can trigger repeated browser connection attempts. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-06 | I | `buildStaffWebSocketUrl` | mitigate | Never log generated URLs; encode token with URLSearchParams. |
| T-13-07 | D | Reconnect loop | mitigate | Use capped exponential backoff with jitter and cancel timers on auth clear. |
| T-13-08 | T | Message parser | mitigate | Validate `{type, payload}` shape before exposing events; ignore malformed frames without throwing. |
| T-13-09 | E | Staff app integration | mitigate | Depend on existing `STAFF_ROLES` login gate and backend enforcement from Plan 13-01; do not connect for unauthenticated sessions. |
</threat_model>
