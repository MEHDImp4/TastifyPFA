<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser URL -> ASGI middleware | Query-string token is untrusted input and can be missing, malformed, expired, or for a non-staff user. |
| ASGI middleware -> Django ORM | Token user ID lookup crosses from async socket code into synchronous database access. |
| Backend code -> channel layer -> socket | Internal backend events become public JSON frames delivered to staff browsers. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | S | `JWTAuthMiddleware` | mitigate | Validate Simple JWT access tokens with Simple JWT `AccessToken`; reject missing/invalid tokens. |
| T-13-02 | E | `StaffConsumer.connect` | mitigate | Enforce backend staff-role gate for `GERANT`, `SERVEUR`, and `CUISINIER`; reject `CLIENT` even if frontend route guards fail. |
| T-13-03 | I | WebSocket query string | mitigate | Do not log token values; tests and helper output must avoid printing full WebSocket URLs. |
| T-13-04 | D | Async consumer | mitigate | Use `database_sync_to_async` in middleware and avoid ORM work in `StaffConsumer` after authentication. |
| T-13-05 | T | Channel-layer event dispatch | mitigate | Use internal event type `staff.event` and public `event_type` mapped to outbound `type` to prevent arbitrary handler dispatch. |
</threat_model>
