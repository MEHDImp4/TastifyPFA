    1. Create `backend/apps/menu/views.py`:
    ```python
    from rest_framework import viewsets, status
    from rest_framework.permissions import IsAuthenticated
    from rest_framework.response import Response

    from apps.users.permissions import IsGerant
    from .models import Categorie
    from .serializers import CategorieSerializer


    class CategorieViewSet(viewsets.ModelViewSet):
        serializer_class = CategorieSerializer

        def get_permissions(self):
            """GERANT: full CRUD. All authenticated users: read-only (per D-05)."""
            if self.action in ('list', 'retrieve'):
                return [IsAuthenticated()]
            return [IsAuthenticated(), IsGerant()]

        def get_queryset(self):
            """D-06: Non-GERANT users only see active categories."""
            user = self.request.user
            if user.is_authenticated and user.role == 'GERANT':
                return Categorie.objects.all().order_by('ordre_affichage', 'nom')
            return Categorie.objects.active().order_by('ordre_affichage', 'nom')

        def destroy(self, request, *args, **kwargs):
            """D-07: Soft delete — sets est_active=False instead of hard-deleting."""
            instance = self.get_object()
            instance.delete()
            return Response(status=status.HTTP_204_NO_CONTENT)
    ```

    Key implementation notes:
    - `get_permissions()` returns `[IsAuthenticated()]` for list/retrieve (read-only for all auth users).
    - `get_permissions()` returns `[IsAuthenticated(), IsGerant()]` for create/update/partial_update/destroy.
    - `get_queryset()` checks `user.role == 'GERANT'` (string comparison matching the TextChoices value).
    - `destroy()` calls `instance.delete()` which triggers the soft-delete override in the model.
    - Serializer context is automatically passed by ModelViewSet, so `use_url=True` on ImageField
      generates absolute URLs.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → POST /api/categories/ | Untrusted multipart data (nom, image) crosses here |
| client → DELETE /api/categories/{id}/ | Must not be callable by non-GERANT |
| get_queryset → DB | Non-GERANT must never receive inactive category rows |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | Spoofing | `CategorieViewSet.get_permissions()` | mitigate | `IsGerant` permission class enforced on all write actions (create, update, partial_update, destroy). Non-GERANT roles receive HTTP 403. Unauthenticated requests receive HTTP 401 via global `IsAuthenticated` default. |
| T-04-06 | Information Disclosure | `CategorieViewSet.get_queryset()` | mitigate | Non-GERANT users receive only `Categorie.objects.active()` queryset — inactive categories are excluded at the ORM level before serialization. Direct GET by ID to an inactive category returns HTTP 404 (queryset filter hides the object). |
| T-04-07 | Tampering | `ImageField` multipart upload | mitigate | DRF `ImageField` with Pillow validates MIME type and file integrity before saving. `upload_to='categories/'` restricts destination path. Files outside this path cannot be written by the API. |
| T-04-08 | Tampering | `destroy()` override | mitigate | `destroy()` calls `instance.delete()` which executes soft delete (sets `est_active=False`). Hard delete path from `super().destroy()` is never called, preventing irreversible data loss. |
| T-04-09 | Elevation of Privilege | `get_permissions()` action check | mitigate | Actions list is explicit: only `('list', 'retrieve')` are read-only. Any new custom action added to the ViewSet will default to requiring `IsGerant` (fail-secure). |
</threat_model>
