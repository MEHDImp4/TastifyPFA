    - Test 1: Empty command list returns empty similarity dict.
    - Test 2: Given a mocked list of `CommandeLigne` where Plats A and B are ordered together often, `compute_similarities` returns B in A's recommendations.
    - Test 3: Celery task `update_recommendations` calls `compute_similarities` and sets the result in `django.core.cache.cache` with key `plat_similarities`.
  </behavior>
  <implementation>
    1. Append `scikit-learn>=1.4.1` and `pandas>=2.2.1` to `app/backend/requirements.txt`.
    2. Ensure `app/backend/apps/menu/ml/__init__.py` exists.
    3. In `recommender.py`, use `pandas` to pivot `CommandeLigne` data into a user-item matrix (rows: `commande_id`, cols: `plat_id`, values: `quantite`). Use `TruncatedSVD` and `cosine_similarity` from `sklearn` to compute similarities and map back to a dictionary of `{plat_id: [similar_plat_ids]}`.
    4. Handle the "cold start" or low-variance case (e.g., fewer than 2 plats ordered) gracefully by returning an empty dict.
    5. In `tasks.py`, create a `@shared_task` named `update_recommendations` that fetches data from `CommandeLigne.objects.values(...)`, passes it to the recommender, and saves the output to Django's cache.
    6. Mock the DB or use actual models in `test_recommender.py` to verify the logic.
  </implementation>
