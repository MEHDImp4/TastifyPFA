## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-13 | Tampering | A SERVEUR could open DevTools and force-render the button on a non-owned order | accept | Front-end UX hide is not a security boundary. Backend (Plan 04) returns 403 — the operation fails server-side. The optimistic UI update will then roll back when error is set. |
| T-16-14 | Denial of Service | Rapid double-click on the fire button triggers two PATCH calls | mitigate | `disabled={isFiring}` prevents double-fire while the request is pending. Backend is idempotent for the EN_COURS->EN_CUISINE transition because the second PATCH would arrive with statut already EN_CUISINE; signal pre/post_save guard requires `prev == EN_COURS` so the orchestrator only runs once. Even without the disable, no double-orchestration occurs. |
